# esp-idf
- When reading HTTP response bodies with the ESP-IDF HTTP client, use `esp_http_client_open()` + `esp_http_client_fetch_headers()` + `esp_http_client_read()` instead of `esp_http_client_perform()`, because `perform()` consumes the response body internally and `read()` returns 0 bytes. Confidence: 0.90
- When using `esp_http_client_set_post_field()` for POST requests, set `config.method = HTTP_METHOD_POST` and use `esp_http_client_open(client, body_len)` + `esp_http_client_write(client, body, body_len)` instead of `esp_http_client_perform(client)`, similarly because `perform()` consumes the body. Confidence: 0.80
- Always add `driver` to `PRIV_REQUIRES` in CMakeLists.txt when using the legacy `driver/i2c.h`. For new `driver/i2c_master.h`, use `esp_driver_i2c`. Confidence: 0.80
- ESP-IDF uses `app_main()` (not `setup()`/`loop()`), and `app_main()` must block with `while(1) { vTaskDelay(...); }` or spawn a FreeRTOS task — if it returns, the main task is deleted and the program ends. Confidence: 0.85
- In ESP-IDF, suppress verbose WiFi/I2C logs by calling `esp_log_level_set("*", ESP_LOG_WARN)` and `esp_log_level_set("FL_CLIENT", ESP_LOG_INFO)` at the start of `app_main()`. Confidence: 0.70
- For I2C on ESP-IDF v6.x, use the legacy `driver/i2c.h` with `i2c_param_config()` + `i2c_driver_install()` when the new `driver/i2c_master.h` driver has pin conflict issues on ESP32. Confidence: 0.70
- For TFLite Micro on ESP32, allocate at minimum a 32KB tensor arena and 32KB task stack to prevent memory corruption (LoadProhibited / Guru Meditation) during TFLite interpreter initialization. Confidence: 0.80
- Validate downloaded TFLite models by checking the first byte is 0x1c (flatbuffer magic) and size is at least 2000 bytes to avoid loading error responses or truncated data. Confidence: 0.85
- When uploading 332 flattened weights from ESP32 to the FL server, the server must reshape them from a flat array into nested format [W1, b1, W2, b2, W3, b3] using numpy reshape before passing to the TFLite model generator, otherwise TensorFlow will throw shape mismatch errors. Confidence: 0.85
- When the FL server serves TFLite models to ESP32 clients, always regenerate the `.tflite` file on each download request using the latest global weights, rather than serving a cached file, to ensure the downloaded model matches the most recent FedAvg result. Confidence: 0.75
